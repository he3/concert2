## Objective

Create the Claude Code skill command files in `templates/.claude/commands/`. Each command file maps a `/concert:<name>` slash command to the corresponding agent definition. When a user types `/concert:status` in Claude Code, Claude loads this command file and follows its instructions to load the full agent definition.

## Detailed Instructions

Each Claude Code command file follows this exact template. Copy it exactly for each command, replacing only the `<agent-name>` and `<description>` placeholders.

### Command file template

```markdown
<!-- AUTO-GENERATED BY CONCERT v0.1.0 — DO NOT EDIT. This file is managed by Concert and will be overwritten on `concert update`. -->

# concert:<command-name>

<description>

## Instructions

1. Read the full agent definition: `docs/concert/agents/concert-<agent-name>.md`
2. Follow the execution flow defined in that agent.
3. Read `docs/concert/state.json` for current mission state.
4. Follow the agent's operating principles and boundaries exactly.
```

### Command mappings

| Command file | Agent | Description |
|-------------|-------|-------------|
| init.md | concert-init | Start a new mission — interactive interview to create vision |
| plan.md | concert-analyst, concert-architect, concert-designer, or concert-planner | Plan the next stage — auto-selects the appropriate consultant agent based on pipeline state |
| run.md | concert-runner | Execute the current phase — runs the code quality loop for each task |
| review.md | concert-reviewer | Review the current stage draft — two-phase review (user changes, then reviewer concerns) |
| accept.md | (direct action) | Accept the current stage draft — copy to *-SPEC.md and advance pipeline |
| status.md | concert-status | Check current mission status — cold-start recovery command |
| continue.md | concert-continue | Continue work — resume after crashes, timeouts, or cross-environment handoffs |
| debug.md | concert-debug | Debug a failure — scientific-method debugging with hypothesis testing |
| verify.md | concert-verifier | Verify completed work — check implementation against requirements |
| quick.md | concert-quick | Run a quick task — skip the full pipeline for small changes |
| push.md | concert-push | Push for handoff — run `npx @he3-org/concert push` (this is a CLI command, not an agent) |
| restart.md | (direct action) | Restart the current stage — discard draft and re-run the consultant |
| replan.md | (direct action) | Replan from a specific stage — mark downstream stages for re-run |

### Special cases

**plan.md** — This command auto-selects the appropriate agent based on pipeline state. Its instructions should be:
1. Read state.json to determine current pipeline state
2. If vision is accepted and requirements are pending: read `docs/concert/agents/concert-analyst.md`
3. If requirements are accepted and architecture is pending: read `docs/concert/agents/concert-architect.md`
4. If architecture is accepted and UX is pending: read `docs/concert/agents/concert-designer.md`
5. If all planning stages are accepted and tasks are pending: read `docs/concert/agents/concert-planner.md`
6. If the user provides an explicit stage argument, use that stage's agent regardless of pipeline state

**accept.md** — This is a direct action, not a full agent. Its instructions should be:
1. Read state.json to determine current stage
2. Read the draft document for the current stage
3. Copy it to `docs/concert/*-SPEC.md` (VISION.md -> VISION-SPEC.md, etc.)
4. Update state.json: `pipeline.<stage> = "accepted"`, advance `stage` to next
5. Update WIP PR body
6. Commit all changes
7. Output next steps

**restart.md** — Direct action:
1. Read state.json to determine current stage
2. Update state.json: `pipeline.<stage> = "pending"`
3. Commit state.json
4. Output: "Stage reset. Run `/concert:plan` to re-run."

**replan.md** — Direct action:
1. Accept stage argument from user
2. Mark all stages after the specified stage as "pending" in state.json
3. Re-run the consultant agent for the specified stage
4. After acceptance, re-run all downstream consultants

**push.md** — This is a CLI command, not an agent. The command file should tell the user to run `npx @he3-org/concert push` in their terminal instead.
