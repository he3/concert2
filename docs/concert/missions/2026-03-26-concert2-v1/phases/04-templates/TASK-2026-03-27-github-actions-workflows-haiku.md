## Objective

Create the GitHub Actions workflow YAML files that ship with Concert. These are CI/CD workflows that run on GitHub Actions — they are separate from Concert's orchestration workflows (which are markdown files read by agents). There are two workflows: a CI test gate that runs tests on every push to a mission branch, and a version check that periodically checks for Concert updates.

## Files

- `templates/.github/workflows/concert-ci.yml`
- `templates/.github/workflows/concert-version-check.yml`

## Detailed Instructions

### templates/.github/workflows/concert-ci.yml

This workflow runs the project's test suite on every push to any `concert/*` branch. It acts as the PR merge gate — the PR cannot be merged unless all tests pass.

**File content (write this exact YAML):**

```yaml
# AUTO-GENERATED BY CONCERT v0.1.0 — DO NOT EDIT. This file is managed by Concert and will be overwritten on `concert update`.
name: Concert CI

on:
  push:
    branches:
      - 'concert/**'
  pull_request:
    branches:
      - main

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - run: npm ci

      - run: npm run typecheck
        name: Type check

      - run: npm test
        name: Run tests

      - run: npm run build
        name: Build
```

### templates/.github/workflows/concert-version-check.yml

This workflow runs periodically (weekly) to check if a newer version of Concert is available and creates an issue if an update is needed.

**File content (write this exact YAML):**

```yaml
# AUTO-GENERATED BY CONCERT v0.1.0 — DO NOT EDIT. This file is managed by Concert and will be overwritten on `concert update`.
name: Concert Version Check

on:
  schedule:
    - cron: '0 9 * * 1'  # Every Monday at 9am UTC
  workflow_dispatch:  # Allow manual trigger

jobs:
  check-version:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: '20'

      - name: Check for Concert updates
        run: |
          # Read current version from concert.jsonc
          CURRENT=$(node -e "
            const fs = require('fs');
            const content = fs.readFileSync('concert.jsonc', 'utf-8');
            // Strip comments for JSON.parse
            const stripped = content.replace(/\/\/.*$/gm, '').replace(/\/\*[\s\S]*?\*\//g, '');
            const config = JSON.parse(stripped);
            console.log(config.concert_version);
          ")

          # Check latest version on npm
          LATEST=$(npm view @he3-org/concert version 2>/dev/null || echo "0.0.0")

          echo "Current: $CURRENT"
          echo "Latest:  $LATEST"

          if [ "$CURRENT" != "$LATEST" ] && [ "$LATEST" != "0.0.0" ]; then
            echo "UPDATE_AVAILABLE=true" >> $GITHUB_ENV
            echo "CURRENT_VERSION=$CURRENT" >> $GITHUB_ENV
            echo "LATEST_VERSION=$LATEST" >> $GITHUB_ENV
          fi

      - name: Create update issue
        if: env.UPDATE_AVAILABLE == 'true'
        uses: actions/github-script@v7
        with:
          script: |
            const title = `Concert update available: v${process.env.CURRENT_VERSION} → v${process.env.LATEST_VERSION}`;

            // Check if issue already exists
            const issues = await github.rest.issues.listForRepo({
              owner: context.repo.owner,
              repo: context.repo.repo,
              state: 'open',
              labels: 'concert-update'
            });

            if (issues.data.some(i => i.title === title)) {
              console.log('Issue already exists, skipping');
              return;
            }

            await github.rest.issues.create({
              owner: context.repo.owner,
              repo: context.repo.repo,
              title: title,
              body: `A new version of Concert is available.\n\n**Current:** v${process.env.CURRENT_VERSION}\n**Latest:** v${process.env.LATEST_VERSION}\n\nUpdate by running:\n\`\`\`\nnpx @he3-org/concert update\n\`\`\``,
              labels: ['concert-update']
            });
```
